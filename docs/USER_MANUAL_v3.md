## 系统概述

DuckDB专业数据导入系统是一款专为量化投资和金融分析设计的高性能数据处理工具。系统集成了智能配置管理、AI驱动的异常检测、自适应缓存等先进技术，为用户提供专业、高效、可靠的数据导入和管理解决方案。

### 主要特性

- **🚀 高性能数据处理**: 基于DuckDB的列式存储，支持大规模数据快速处理
- **🤖 智能配置管理**: AI驱动的配置优化，自动检测和解决配置冲突
- **🔍 异常检测与修复**: 多维度数据质量监控，智能异常检测和自动修复
- **💾 自适应缓存**: 基于使用模式的智能缓存策略，提升数据访问效率
- **📊 现代化界面**: 专业的PyQt5界面设计，支持多标签页和数据可视化
- **⚡ 预测性加载**: 机器学习驱动的数据预加载，提前准备常用数据

### 支持的数据源

- **通达信**: 股票、指数、基金等实时和历史数据
- **AKShare**: 开源金融数据接口，覆盖多种金融产品
- **Wind**: 专业金融数据终端接口
- **自定义数据源**: 支持扩展其他数据源

### 支持的数据类型

- **K线数据**: 日线、分钟线、周线、月线
- **实时行情**: 实时价格、成交量、买卖盘
- **基本面数据**: 财务报表、公司基本信息
- **指数数据**: 各类股票指数、行业指数
- **基金数据**: 基金净值、持仓、业绩

1. **下载系统**
   ```bash
   git clone https://github.com/your-org/hikyuu-ui.git
   cd hikyuu-ui
   ```

2. **安装依赖**
   ```bash
   pip install poetry
   poetry install
   ```

3. **启动系统**
   ```bash
   poetry run python main.py
   ```

## 功能详解

### 创建数据导入任务

#### 步骤1: 启动任务创建向导

1. 在任务管理标签页点击"新建任务"按钮
2. 或使用快捷键 `Ctrl+N` (Windows/Linux) 或 `Cmd+N` (macOS)
3. 系统会打开任务创建向导对话框

#### 步骤2: 基本信息配置

**任务基本信息**:
- **任务名称**: 为任务起一个有意义的名称，如"沪深300日线数据"
- **任务描述**: 可选，详细描述任务的用途和目标
- **数据源**: 选择数据来源（通达信、AKShare、Wind等）
- **资产类型**: 选择资产类型（股票、指数、基金、债券等）

**示例配置**:
```
任务名称: 沪深300成分股日线数据
任务描述: 获取沪深300指数成分股的日线K线数据，用于量化策略回测
数据源: 通达信
资产类型: 股票
```

#### 步骤3: 数据范围配置

**标的代码设置**:
- **手动输入**: 直接输入股票代码，多个代码用逗号分隔
- **文件导入**: 从CSV或TXT文件导入代码列表
- **指数成分股**: 选择指数自动获取成分股代码
- **条件筛选**: 根据市值、行业等条件筛选股票

**时间范围设置**:
- **开始日期**: 数据获取的起始日期
- **结束日期**: 数据获取的结束日期（可选，默认为当前日期）
- **数据频率**: 选择数据频率（日线、分钟线、周线、月线）

**示例配置**:
```
标的代码: 从沪深300指数成分股中选择
开始日期: 2020-01-01
结束日期: 2024-01-01
数据频率: 日线
```

#### 步骤4: 执行参数配置

**性能参数**:
- **最大工作线程数**: 控制并发下载的线程数量（建议2-8）
- **批处理大小**: 每批处理的记录数量（建议500-2000）
- **请求间隔**: 两次请求之间的间隔时间（毫秒）
- **超时设置**: 单次请求的超时时间（秒）

**执行模式**:
- **立即执行**: 创建任务后立即开始执行
- **定时执行**: 设置定时执行计划（支持Cron表达式）
- **手动执行**: 需要手动触发执行

**示例配置**:
```
最大工作线程数: 4
批处理大小: 1000
请求间隔: 100ms
超时设置: 30秒
执行模式: 立即执行
```

#### 步骤5: 高级选项配置

**数据质量控制**:
- **启用异常检测**: 自动检测数据异常
- **自动修复**: 对检测到的异常进行自动修复
- **质量阈值**: 设置数据质量的最低要求

**缓存设置**:
- **启用缓存**: 使用智能缓存提高性能
- **缓存过期时间**: 缓存数据的有效期
- **预测性加载**: 启用AI驱动的预测性数据加载

**示例配置**:
```
启用异常检测: 是
自动修复: 是
质量阈值: 95%
启用缓存: 是
缓存过期时间: 1小时
预测性加载: 是
```

### 任务执行与监控

#### 任务状态说明

- **待执行**: 任务已创建但尚未开始执行
- **执行中**: 任务正在执行数据导入
- **已完成**: 任务成功完成所有数据导入
- **已暂停**: 任务被用户手动暂停
- **执行失败**: 任务因错误而停止执行
- **已取消**: 任务被用户取消

#### 实时监控功能

**进度监控**:
- **总体进度**: 显示任务的整体完成百分比
- **当前批次**: 显示正在处理的数据批次信息
- **剩余时间**: 预估任务完成的剩余时间
- **处理速度**: 当前的数据处理速度

**性能监控**:
- **CPU使用率**: 任务执行时的CPU占用情况
- **内存使用量**: 任务执行时的内存占用情况
- **网络流量**: 数据下载的网络流量统计
- **磁盘I/O**: 数据写入的磁盘I/O统计

**质量监控**:
- **成功率**: 成功获取数据的比例
- **错误率**: 数据获取失败的比例
- **异常检测**: 实时检测到的数据异常数量
- **修复率**: 自动修复成功的异常比例

#### 任务控制操作

**基本控制**:
- **启动任务**: 开始执行选中的任务
- **暂停任务**: 暂停正在执行的任务
- **恢复任务**: 恢复已暂停的任务
- **停止任务**: 强制停止正在执行的任务
- **重新执行**: 重新执行已完成或失败的任务

**高级控制**:
- **优先级调整**: 调整任务的执行优先级
- **资源限制**: 动态调整任务的资源使用限制
- **断点续传**: 从上次中断的位置继续执行
- **增量更新**: 只获取新增或更新的数据

### 数据质量管理

#### 异常检测机制

系统采用多维度的异常检测算法，能够自动识别各种数据质量问题：

**1. 数据完整性检测**
- **缺失值检测**: 识别关键字段的缺失数据
- **空值检测**: 检测空字符串或NULL值
- **格式检测**: 验证数据格式是否符合预期

**2. 数据准确性检测**
- **范围检测**: 检测数值是否在合理范围内
- **逻辑检测**: 验证数据之间的逻辑关系
- **一致性检测**: 检测同一实体的数据一致性

**3. 数据时效性检测**
- **时间序列检测**: 检测时间序列数据的连续性
- **更新及时性**: 检测数据更新是否及时
- **历史一致性**: 验证历史数据的一致性

#### 异常处理流程

**1. 异常识别**
```
数据输入 → 多维度检测 → 异常分类 → 严重程度评估 → 异常记录
```

**2. 修复建议生成**
```
异常分析 → AI推理 → 修复策略 → 置信度评估 → 修复建议
```

**3. 自动修复执行**
```
修复建议 → 风险评估 → 自动执行 → 结果验证 → 修复记录
```

#### 修复策略说明

**数据缺失修复**:
- **插值法**: 使用线性插值或样条插值填充缺失值
- **统计填充**: 使用均值、中位数或众数填充
- **模型预测**: 使用机器学习模型预测缺失值
- **历史数据**: 使用历史同期数据填充

**异常值修复**:
- **截断处理**: 将异常值调整到合理范围内
- **平滑处理**: 使用移动平均等方法平滑异常值
- **删除处理**: 删除明显错误的异常值
- **标记处理**: 保留异常值但进行标记

**重复数据修复**:
- **去重处理**: 删除重复的数据记录
- **合并处理**: 将重复记录合并为一条记录
- **保留最新**: 保留时间戳最新的记录
- **人工审核**: 标记为需要人工审核

### 智能配置优化

#### 配置推荐系统

系统内置AI驱动的配置推荐引擎，能够根据历史性能数据和当前环境自动生成优化建议：

**推荐类型**:
- **性能优化**: 提升数据导入速度和吞吐量
- **可靠性优化**: 提高任务执行成功率和稳定性
- **成本优化**: 降低资源消耗和运行成本
- **平衡优化**: 在性能、可靠性和成本之间找到平衡

**推荐依据**:
- **历史性能数据**: 分析过往任务的执行表现
- **系统资源状况**: 考虑当前系统的资源使用情况
- **数据源特性**: 根据不同数据源的特点进行优化
- **任务特征**: 根据任务的数据量、频率等特征调整

#### 配置冲突检测

**资源冲突检测**:
- **CPU资源**: 检测多个任务是否会导致CPU过载
- **内存资源**: 检测内存使用是否超出系统限制
- **网络带宽**: 检测网络请求是否会导致带宽饱和
- **存储空间**: 检测磁盘空间是否足够存储数据

**调度冲突检测**:
- **时间冲突**: 检测定时任务的执行时间是否冲突
- **依赖冲突**: 检测任务之间的依赖关系是否合理
- **优先级冲突**: 检测任务优先级设置是否合理

**自动解决方案**:
- **资源调整**: 自动调整任务的资源分配
- **时间错开**: 自动调整任务的执行时间
- **优先级重排**: 自动调整任务的执行优先级
- **分批执行**: 将大任务拆分为多个小任务

### 任务配置优化

#### 性能优化建议

**1. 合理设置并发参数**
- **小数据量任务**: 线程数2-4，批处理大小500-1000
- **中等数据量任务**: 线程数4-6，批处理大小1000-2000
- **大数据量任务**: 线程数6-8，批处理大小2000-5000

**2. 优化请求频率**
- **通达信数据源**: 建议请求间隔100-200ms
- **AKShare数据源**: 建议请求间隔50-100ms
- **Wind数据源**: 建议请求间隔200-500ms

**3. 合理使用缓存**
- **实时数据**: 缓存时间5-15分钟
- **日线数据**: 缓存时间1-4小时
- **基本面数据**: 缓存时间1-7天

## 故障排除

### 常见问题诊断

#### 任务执行失败

**问题现象**: 任务状态显示"执行失败"，日志中有错误信息

**可能原因**:
1. 网络连接问题
2. 数据源API限制
3. 系统资源不足
4. 配置参数错误

**诊断步骤**:
1. 检查网络连接状态
2. 查看详细错误日志
3. 验证数据源配置
4. 检查系统资源使用情况

**解决方案**:
```
网络问题 → 检查网络连接，配置代理
API限制 → 调整请求频率，联系数据源提供商
资源不足 → 减少并发数，增加系统资源
配置错误 → 检查并修正配置参数
```

#### 数据质量异常

**问题现象**: 异常检测标签页显示大量数据异常

**可能原因**:
1. 数据源数据质量问题
2. 网络传输错误
3. 系统处理异常
4. 配置参数不当

**诊断步骤**:
1. 查看异常详情和类型分布
2. 检查数据源的原始数据
3. 验证网络传输稳定性
4. 检查系统处理逻辑

**解决方案**:
```
源数据问题 → 联系数据源提供商
传输错误 → 启用数据校验，增加重试
处理异常 → 检查系统日志，修复bug
配置不当 → 调整异常检测阈值
```

#### 系统性能问题

**问题现象**: 系统响应缓慢，任务执行时间过长

**可能原因**:
1. 系统资源不足
2. 数据库性能问题
3. 网络带宽限制
4. 配置参数不优

**诊断步骤**:
1. 检查系统资源使用情况
2. 分析数据库查询性能
3. 测试网络带宽和延迟
4. 检查任务配置参数

**解决方案**:
```
资源不足 → 增加CPU、内存资源
数据库慢 → 优化查询，重建索引
网络限制 → 优化网络配置，使用CDN
配置不优 → 使用AI推荐的优化配置
```

### 日志分析

#### 日志级别说明

- **DEBUG**: 详细的调试信息，用于开发和故障排除
- **INFO**: 一般信息，记录系统正常运行状态
- **WARNING**: 警告信息，系统可以继续运行但需要注意
- **ERROR**: 错误信息，系统出现错误但可以恢复
- **CRITICAL**: 严重错误，系统无法正常运行
