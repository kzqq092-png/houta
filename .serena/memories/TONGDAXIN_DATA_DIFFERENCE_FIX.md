## 通达信插件数据差异问题修复

### 问题分析
**数据差异**: 官方统计5,123只A股 vs 插件4,610只A股，差异-513只(-10%)

**分市场差异**:
- 上证: 官方2,272 vs 插件2,291 (+19, +0.8%) ✅ 正常
- 深证: 官方2,851 vs 插件2,319 (-532, -18.7%) ❌ 明显偏少

### 根本原因
1. **A股过滤规则过于严格**: 遗漏了某些A股代码段
2. **深证数据不完整**: 可能遗漏001xxx、004xxx等代码段
3. **缺乏详细统计**: 无法分析被过滤的代码段

### 执行的修复

#### 1. 扩展A股过滤规则
**修改文件**: `plugins/data_sources/stock/tongdaxin_plugin.py`
**修改方法**: `_is_a_stock()`

**原始规则**:
- SH: 600xxx, 601xxx, 603xxx, 605xxx, 688xxx
- SZ: 000xxx, 002xxx, 003xxx, 300xxx

**扩展规则**:
- SH: 600xxx, 601xxx, 603xxx, 605xxx, 688xxx, 689xxx
- SZ: 000xxx, 001xxx, 002xxx, 003xxx, 004xxx, 300xxx

#### 2. 增强统计和日志记录
**修改方法**: `get_stock_list()`

**新增功能**:
- 代码前缀分布统计
- 被过滤代码段分析
- 详细的过滤前后对比
- 更全面的调试信息

#### 3. 预期改善效果
- 深证数据从2,319只增加到接近2,851只
- 总数据量从4,610只增加到接近5,123只
- 减少被误过滤的A股
- 提供更好的调试和分析能力

### 修复内容总结
1. ✅ 扩展A股代码过滤规则
2. ✅ 添加详细的统计和日志记录
3. ✅ 分析被过滤的代码段
4. ✅ 提供更好的调试信息

### 验证方法
重新运行通达信插件，检查：
1. 总数据量是否更接近5,123只
2. 深证数据是否更接近2,851只
3. 日志中的代码前缀分布统计
4. 被过滤代码段的分析结果
