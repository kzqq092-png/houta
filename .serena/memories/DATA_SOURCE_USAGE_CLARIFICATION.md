## 数据源插件使用情况澄清 - 关键发现

### 问题澄清
用户问为什么系统能获取到上万条数据，尽管通达信插件有10000限制。

### 关键发现
**用户实际使用的是东方财富插件，而不是通达信插件！**

### 系统配置分析
**文件**: core/services/unified_data_manager.py 第213行
```python
'stock': ['eastmoney', 'sina', 'tonghuashun']
```
- 东方财富(eastmoney)是股票数据的**首选数据源**
- 通达信插件不在默认优先级列表中

### 两个插件的对比

#### 通达信插件 (TongdaxinStockPlugin)
- **文件**: plugins/data_sources/stock/tongdaxin_plugin.py
- **限制**: 第1178、1191行有`min(count, 10000)`限制
- **数据量**: SH(2,272) + SZ(2,851) = 5,123条
- **状态**: 存在但非默认使用

#### 东方财富插件 (EastMoneyStockPlugin)  
- **文件**: plugins/data_sources/stock/eastmoney_plugin.py
- **限制**: 无硬性限制
- **数据量**: 一次获取5000条，支持分页
- **包含板块**: 沪市A股 + 科创板 + 深市A股 + 创业板
- **状态**: 系统默认首选数据源

### 数据量对比
- **通达信**: ~5,123条 (仅主板A股)
- **东方财富**: ~6,600+条 (包含科创板、创业板等)

### 为什么用户看到上万条数据
1. 系统默认使用东方财富插件
2. 东方财富包含更多板块（科创板、创业板）
3. 东方财富无10,000条限制
4. 可能包含B股、ST股等特殊股票
5. 数据更新频率不同

### 解决方案
- **如果使用东方财富**: 无需修复通达信限制
- **如果使用通达信**: 需要修复10,000限制 + 修改数据源优先级
